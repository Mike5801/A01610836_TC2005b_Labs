Preguntas del laboratorio

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

    2. La World Wide Web son las páginas que visualizamos cuando estamos conectados en línea dentro de un dispositivo y el Internet es la red de computadoras conectadas en la cual trabaja la world wide web.

  2. ¿Cuáles son las partes de un URL?

    3. Consiste de cuatro partes:

    • Protocolo: Usualmente son HTTP o HTTPS, y son protocolos de comunicación
    • Dominio: Nombre usado para identificar la dirección ip del recurso
    • Ruta: Especifica la ubicación del recurso dentro del servidor
    • Parametros: Datos adicionales para identificar o filtrar el recurso en el servidor
  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • GET: Método para pedir datos de un recurso sin cambiar su estado
    • HEAD: Método similar a GET, con la diferencia de que no devuelven el cuerpo del mensaje como respuesta
    • POST: Método para enviar datos a un servidor para crear un recurso
    • PUT: Método para actualizar un recurso existente en el servidor
    • PATCH: Método para aplicar modificaciones parciales a un recurso
    • DELETE: Método para eliminar un recurso específico
    >
  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    5. El método POST, ya que al enviar este tipo de información es necesario que el cuerpo esté encriptado por cuestiones de seguridad

  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    6. El método GET, ya que cada URL tiene una dirección de IP a la que está conectada y para acceder al a página es necesario obtener esa dirección IP

  6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    7. Que corrió exitosamente y no hay errores

  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    8. No, ya que habla de que el usuario intenta buscar una URL que no existe. Sin embargo, también puede tratarse de un elemento que sí existe en el servidor, pero se muestra este error para ocultar la existencia del recurso en lugar de mostrar el error 403 que habla sobre no tener los permisos suficientes para acceder

  8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    9. Sí, ya que se trata de un error dentro del servidor

  9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    10. Significa que un atributo está obsoleto por otros más actuales, sin embargo, no implica que no se puedan usar, ya que HTML tiene la característica de tener una compatibilidad con las versiones anteriores

  10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    11. Una de las principales diferencias entre HTML 4 y HTML 5, es que en su versión anterior no suportaba archivos multimedia, mientras que en el más actual sí lo permite. Además, en su versión nueva, proporciona nuevos elementos como etiquetas semánticas que proporcionan estructura a la página como nav, header, footer, entre otros.

  11. ¿Qué componentes de estructura y estilo tiene una tabla?

    12. La estructura de las tablas tiene celdas representadas con la etiqueta td, filas representadas con la etiqueta tr y títulos de celdas representadas con la etiquta th. También es posible agrupar la tabla en diferentes partes utilizando las etiquetas thead, tbody y tfoot. En cuanto a la parte de estilos, las tablas tienen etiquetas como caption, colgroup y col.

  14. ¿Cuál es el ciclo de vida de los sistemas de información?
    1. Planificación: encontrar el alcance del problema y determinar las diferentes soluciones al igual que los recursos, costos, tiempo, beneficios, entre otros
    2. Análisis del sistema y requisitos: considerar los requisitos funcionales del proyecto o de la solución. También analiza las necesidades de los usuarios para cumplir con sus expectativas
    3. Diseño del sistema: describe detalladamente las especificaciones necesarias, características y operaciones que deberá de satisfacer el sistema propuesto.
    4. Desarrollo: inicia la producción.
    5. Integración y testing: implica la integración del sistema y el testeo de sus programas y procedimientos. Es normalmente llevado a cabo por un Quality Assurance profesional para determinar si se lograron los objetivos propuestos en el inicio del proyecto
    6. Implementación: fase en donde el programa comienza a ser usado y se transfieren los datos y componentes del sistema viejo al nuevo.
    7. Operaciones y mantenimiento: los usuarios afinan el sistema para mejorar el rendimiento, agregar nuevas herramientas o proponer nuevos requisitos.
  15. ¿Cuál es el ciclo de desarrollo de sistemas de información?

    16. Está dentro del ciclo de vida de los sistemas de información y es donde comienza a escribirse el código y a contruir la aplicación con base en los diseños y especificaciones escritas en las fases anteriores.
